# Contributing to OpenCompliance Conformance

This repo holds public conformance vectors and expected outputs.

## Ground rules

- Every vector must point to a public example or public spec.
- Do not write fake tests for software that does not exist.
- Keep expected outputs deterministic and reviewable.
- Negative cases are better than vague ambition.

## Before opening a PR

- State which public example or spec the vector depends on.
- Update expected outputs if the example changes.
- Explain whether the change is additive or breaking.
- Keep the corpus synthetic and safe to publish.
